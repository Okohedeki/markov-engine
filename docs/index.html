markov-engine

A storage-agnostic knowledge engine — a CLI and a library. Give it a URL; it extracts, embeds, clusters into a Chain, grows the Chain from the web, and writes artifacts. Agents can run it as a tool.

# save a link → it clusters into a Chain
markov ingest https://youtu.be/UF8uR6Z6KLc
# grow the Chain — discover related sources
markov grow 1 --hops 2 --budget 10
# synthesize an article across the Chain
markov generate 1 --type article
  • Pluggable Store — local SQLite ships by default
  • Extraction: articles, PDFs, YouTube/TikTok/Reddit/X, audio
  • Runs on Anthropic — or a local model (validated on Qwen-3B)

How the product works

How a save becomes a living Chain

The same four steps run whether you use the CLI, a script, or the app.

01 · SOURCE

Save

A link is extracted to text — transcribing video/audio when needed.

02 · TOPIC

Understand

An LLM pulls a summary, entities, and relationships; the summary is embedded.

03 · CHAIN

Cluster

Similar Topics auto-combine into one Chain by embedding distance — or seed a new one.

04 · GROW

Research

The Chain keeps pulling in new related sources, bounded by a relevance-decay floor.

Built to be embedded — and proven

Validated end-to-end on a real YouTube video, including a depth-6 growth run.

hop_depth

Tunable reach

Growth fans out across adjacency hops; a relevance-decay floor stops Chains from ballooning.

Store

Any backend

One small interface. SQLite locally; Postgres + pgvector in the cloud product.
