Open-source knowledge engine ·MIT ·CLI + library

Save once. The subject keeps researching itself.

Give Markov a link — an article, a PDF, a YouTube video — and it pulls out the ideas, files them under the subject they belong to, and builds a living Chain. Then it keeps going: each Chain searches the web for new related sources on its own, and writes the article when you ask.

Get the engine See the app it powers

It does the work a diligent researcher would — and then it doesn't stop.

Markov reads the page — transcribing video and audio when it has to — and pulls out a summary plus the people, places, and ideas inside it. It embeds that and files the result under the existing subject it most resembles, or starts a new Chain when nothing fits. One save becomes a place your knowledge accumulates instead of a bookmark you forget.

Then it keeps working between your sessions. On a schedule, each Chain searches the web for new related sources and folds them in — bounded by a relevance floor, so a Chain deepens around its subject instead of sprawling into everything. Ask, and it writes the article, newsletter, or brief, synthesized across every source the Chain has gathered.

Storage-agnostic. One small Store interface — SQLite ships by default; plug in Postgres, a vector DB, anything.
Reads everything. Articles, PDFs, YouTube · TikTok · Reddit · X, audio.
Runs on your model. Anthropic — or a local GGUF. Proven end-to-end on a real YouTube video at growth depth 6, with a 3B model doing extraction and writing. No API keys required.
Built for agents. A clean CLI and an async library your own code can drive.
§ The pipeline
01 — Source

Save

A link becomes clean text, transcribing media when needed.

02 — Topic

Understand

A model extracts a summary, entities, and relationships; the summary is embedded.

03 — Chain

Cluster

Similar subjects auto-combine into one Chain by embedding distance — or seed a new one.

04 — Grow

Research

The Chain keeps pulling in new sources over time, deepening instead of ballooning.

§ One engine, two ways to use it
Open source · MIT

markov-engine

The engine itself — a CLI and an async library. Run it on your laptop, embed it in your product, or hand it to an agent as a tool. The brain is open; the backend is yours.

  • Pluggable Store; local SQLite by default
  • ingest · grow · generate from the command line
  • Anthropic or a local model
